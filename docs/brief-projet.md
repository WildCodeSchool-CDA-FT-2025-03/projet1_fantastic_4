# **Brief de Projet : Home Center**

## **Context**

Dans le cadre de votre formation en tant que Développeur Concepteur d'Applications, vous allez travailler en équipe de 2 à 3 personnes sur un projet pratique. Ce projet a pour objectif de mettre en application les compétences techniques que vous avez acquises, notamment :

* GraphQL
* SQLite
* TypeORM
* React
* Gestion de version avec Git

## **Objectif**

Votre mission est de développer un **Minimum Viable Product (MVP) de Home Center**, une application permettant aux utilisateurs de naviguer et de consulter un catalogue de livres, d’albums de musiques, de jeux vidéo et de films. Vous aurez à disposition un dataset prédéfini contenant des informations sur ces éléments.

Votre application devra proposer des fonctionnalités pertinentes selon votre propre vision du produit. Vous devrez justifier vos choix fonctionnels et techniques.

## **Contraintes et Attentes**

* **Aucune fonctionnalité d'upload ou d'authentification** n'est requise.
* Vous devez implémenter un système de navigation fluide et ergonomique.
* L'interface doit être claire et intuitive.
* Vous devez utiliser une API (GraphQL) pour manipuler les données.
* La base de données SQLite et TypeORM doivent être intégrés.
* Le code doit être versionné avec Git (utilisation de branches, commits clairs, etc.).
* Le projet doit être documenté (README.md expliquant l'installation, l'usage, les choix techniques, etc.).

## **Fonctionnalités Attendues (Suggestions)**

Vous avez la liberté de choisir les fonctionnalités que vous estimez les plus pertinentes. Voici quelques idées :

* **Catalogue interactif** : Liste des livres, musiques, jeux vidéos et films avec filtres et tri.
* **Système de favoris** : Permettre aux utilisateurs de marquer des éléments en favoris.
* **Système de notation** : Ajouter une note sur un livre ou un film, musiques, jeux.
* **Recommandations** : Proposer des films/livres/jeux/musiques similaires selon les choix de l'utilisateur.
* **Mode sombre** : Interface adaptée en fonction des préférences de l'utilisateur.

## **Méthodologie de Travail**

### **Organisation**

* Formation d'équipes de 3 à 4 personnes.
* Chaque développeur doit réaliser un CRUD complet avec mise en place des tables associées (Fullstack)
* Utilisation d'un outil de gestion de projet (Trello, Notion, GitHub Projects, etc.).

### **Phases du projet**

1. **Idéation** : Définir les fonctionnalités prioritaires et l'architecture du projet.
2. **Conception** : Établir le schéma de la base de données et l'architecture du code.
3. **Développement** : Implémentation des fonctionnalités en suivant une approche itérative.
4. **Tests et ajustements** : Réalisation de tests unitaires et fonctionnels.
5. **Finalisation et présentation** : Mise au propre du projet et documentation.

## **Mise en Compétition Friendly**

Afin de stimuler l'engagement et l'innovation, chaque équipe présentera son projet en fin de session. Une "Appel à projet" sera organisé, et les critères suivants seront évalués :

* **Pertinence et originalité des fonctionnalités choisies** (30%)
* **Qualité de l'interface utilisateur et expérience utilisateur (UX/UI)** (25%)
* **Qualité et structuration du code (Clean Code, bonnes pratiques)** (25%)
* **Documentation claire et complète** (10%)
* **Démonstration et pitch du projet** (10%)

Une récompense symbolique (badge) sera remise à l'équipe ayant présenté le projet le plus abouti.

## **Livrables Attendus**

* Code source sur un dépôt GitHub.
* Documentation technique et fonctionnelle (README.md).
* Démonstration en fin de projet (pitch et démo de l'application).
